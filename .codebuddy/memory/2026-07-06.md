## 生产环境 CSP 修复（媒体无法加载）
- 用户反馈：打包后插件能扫描但图片/音乐/视频无法加载。
- 根因：`tauri.conf.json` 的 CSP 缺少 `connect-src` 指令，回退到 `default-src 'self'`（生产环境 self = https://tauri.localhost），不包含 asset 协议域。WebView2 加载媒体时先发 HTTP 请求（range request），受 connect-src 约束 → 被拦截。开发模式 CSP 不生效所以正常。
- 修复：CSP 添加 `connect-src 'self' asset: https://asset.localhost http://asset.localhost`；img-src/media-src 补充 `blob:` 和 `http://asset.localhost`。
- 注意：此修改在 `tauri.conf.json` 中，需 `pnpm tauri build` 重新打包才生效（仅 `pnpm build` 不够）。

## Bug 修复：关闭不退出 / 终端刷歌词位置 / 多个托盘图标无响应
- 根因三合一：托盘模式启用后关闭主窗口只隐藏（进程常驻）；`LyricsWidget.tsx` 的 `setInterval(savePos, 2000)` 无条件每 2 秒调用 `save_lyrics_widget_position` 并 eprintln，导致终端一直刷 `[Lyrics] 位置已保存: (100,100)`；重复启动堆积多个进程→多个托盘图标且交互混乱。
- 修复 1（前端 `src/core/lyrics/LyricsWidget.tsx`）：保存前比较 `lastSavedPos`，位置无变化（阈值<1px）则跳过保存；挂载时通过 `get_lyrics_widget_position`（返回元组序列化为数组 `[x,y]`）初始化 lastSavedPos，避免首启重复写默认 (100,100)。注意 Rust 元组 `(f64,f64)` 序列化为数组而非 `{x,y}` 对象。
- 修复 2（`src-tauri/src/main.rs` 单实例守卫）：用 `UdpSocket::bind("127.0.0.1:45991")` 做跨进程互斥；主实例后台监听聚焦请求（收到即 show+focus main），重复实例发 `focus` 后 `std::process::exit(0)`。防止重复启动产生多个托盘图标。
- 修复 3（`main.rs` on_window_event）：托盘模式关闭主窗口时同步隐藏 `lyrics-widget` 悬浮窗；非托盘模式关闭主窗口时 `destroy()` 歌词悬浮窗，确保进程真正退出（否则歌词 webview 窗口常驻使进程不退出）。
