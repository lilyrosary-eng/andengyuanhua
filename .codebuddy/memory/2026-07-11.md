# 2026-07-11 工作日志

## 修复 3303 个 TypeScript 错误 → 0 个

### 根本原因
- `node_modules` 中 pnpm 的 symlink/junction 全部损坏：`fs.statSync('node_modules/react')` 返回 "UNKNOWN: unknown error"
- Windows 系统 reparse point 不可跟随（symlink 和 junction 都不行），导致 TypeScript 找不到 `@types/react`/`@types/react-dom`（TS2688），级联出 3303 个错误

### 修复步骤
1. **创建 `scripts/fix-node-modules.mjs`**：扫描 `node_modules/` 中所有断掉的 symlink，从 `.pnpm` store 拷贝真实目录替换（共修复 56 个）
2. **`.npmrc` 添加 `node-linker=hoisted`**：未来 pnpm install 使用扁平模式避免 symlink
3. **tsconfig.json 修复**：根和 plugins 的 `types` 从 `["react","react-dom"]` 改为 `[]`
4. **Radix UI 类型修复**（context-menu.tsx/slider.tsx/switch.tsx）：添加 className/children/onClick/asChild 到类型注解，JSX 用 `as any` 别名
5. **RichTextEditor.tsx 修复**：`type Editor = NonNullable<ReturnType<typeof useEditor>>` 替代 `@tiptap/core` 导入；添加 handleKeyDown/onUpdate 回调类型注解

### 结果
- `npx tsc --noEmit` 返回 exit code 0，零错误
