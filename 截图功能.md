# 截图功能说明

> 本文完整说明安灯花园（tauri-best）的截图能力：触发方式、整体架构、运行流程、核心能力、后端（Rust）命令清单、前端关键文件、性能设计、与微信/QQ 的对比、后续计划，并在「详细代码与引用来源」一节给出关键实现代码（每段以 `//<源文件>:<行号> 实现功能：<说明>` 标注来源）。文末附引用，便于逐项提问。

## 一、功能概述

截图是系统的「全局能力」之一，目标是做到「微信式体验」：任何时刻（包括应用失焦、最小化到托盘）按全局热键即可秒开截图覆盖层，框选 / 点窗 / 长截图后做标注，保存到剪贴板与中转站，并可直接导入当前笔记。

当前已实现的能力主干：

- 系统级全局热键触发（默认 `Ctrl+Shift+S`，可在设置面板自定义并持久化）。
- 覆盖整虚拟桌面（多显示器）捕获。
- 自由拖拽框选、悬停高亮窗口并单击整窗截取、窗口长截图（捕获完整窗口内容）。
- 标注工具：画笔、橡皮、矩形、箭头、文字、撤销、清空。
- 复制 / 存中转站 / 导入当前笔记联动。
- 毫秒级保存（原生 RGBA 直通，不走 PNG 编码往返）。

## 二、触发方式

1. 全局热键：Rust 侧 `register_screenshot_shortcut`（`src-tauri/src/screenshot.rs:1039`）通过 Tauri `global_shortcut` 注册，应用失焦也能触发，派发 `open-screenshot` 事件到主窗口。
2. 前端监听：`src/App.tsx:50` 的 `useEffect` 监听 `open-screenshot` → 调用 `startScreenshot`（`src/App.tsx:36`）。
3. `startScreenshot` 先记录当前笔记页 id（`store_screenshot_note_id`，供「导入当前笔记」用），再 `invoke('start_screenshot')`（`src/App.tsx:42`）。
4. 热键自定义：设置面板 `src/core/settings/GlobalSettingsPanel.tsx` 提供「全局截图」改键，调用 `set_screenshot_shortcut`（见下文命令清单），持久化到 `app_data/screenshot_shortcut.json`。

> 说明：主窗口全程不隐藏。软件运行（含最小化 / 托盘）时随时可截图；主窗口可见时会被截入画面，这是为「随时可截」主动做的取舍，已在 UI 注释中说明（`src/App.tsx:33`）。

## 三、整体架构（四层）

截图由四层协作，覆盖层不加载主应用，以确保「秒开」：

1. **后端命令层**（`src-tauri/src/screenshot.rs`）：负责实际捕获（`BitBlt`）、窗口枚举、原生裁剪、保存落盘，以及覆盖窗的创建 / 显隐。模块在 `src-tauri/src/lib.rs:3` 以 `pub mod screenshot;` 注册。
2. **覆盖窗（独立轻量入口）**：Rust 在 setup 预创建名为 `screenshot-overlay` 的透明覆盖窗（`src-tauri/src/screenshot.rs:101`），以独立 HTML `screenshot-overlay.html` 加载，仅引入 `src/overlay-screenshot.ts`，**不加载主应用**（`src/main.tsx:42` 注释说明）。这样彻底避免「整站 JS 包解析」带来的数秒延迟。
3. **覆盖窗入口**（`src/overlay-screenshot.ts`）：监听 `screenshot-start` 事件，拿到覆盖窗原点 `ox/oy`、`scale`、窗口列表 `windows`、`noteId` 后，调用 `read_screenshot` 取出预览 JPEG，构造 `ScreenshotOverlay`。
4. **交互 UI**（`src/components/ScreenshotOverlay.tsx`）：选择态（框选 / 点窗 / 长截图）与编辑态（标注工具栏）的全部逻辑，约 865 行。

## 四、完整运行流程

1. 热键触发 → 主窗口 `startScreenshot`（`src/App.tsx:36`）→ `start_screenshot`（`src-tauri/src/screenshot.rs:381`）。
2. `start_screenshot` 同步捕获：取覆盖窗真实矩形作为捕获区（`BitBlt` 整窗，见 `capture_region` `src-tauri/src/screenshot.rs:249`），生成两份图 —— 原生物理 RGBA（`Shot.raw`，保存用）+ 降采样 JPEG 预览（`Shot.preview`，秒开显示用）；同时枚举窗口（`list_windows` `src-tauri/src/screenshot.rs:157`）。
3. 经事件 `screenshot-start` 把 `{ox, oy, scale, windows, noteId}` 推给覆盖窗（`src-tauri/src/screenshot.rs:508`），并 `show` + `set_focus`。
4. 覆盖窗 `overlay-screenshot.ts` 收到事件 → `read_screenshot`（`src-tauri/src/screenshot.rs:516`）取预览 JPEG → 渲染 `ScreenshotOverlay`。
5. **选择态**：拖拽框选 / 单击窗口 / 长截图入口。选区确认（Enter 或松手）→ `commitCrop`（`src/components/ScreenshotOverlay.tsx:179`）建立选区底图并换算原生坐标。
6. **编辑态**：画笔 / 橡皮 / 矩形 / 箭头 / 文字标注，支持撤销、清空。
7. **保存**：`save`（`src/components/ScreenshotOverlay.tsx:530`）经 `composeBytes`（`src/components/ScreenshotOverlay.tsx:495`）合成原生 RGBA 字节，交给 `save_screenshot`（`src-tauri/src/screenshot.rs:597`）写入剪贴板 + 落盘中转站，返回 `localimg://` 引用。
8. 若触发时正处于某篇笔记页，覆盖窗转发 `screenshot-note-import` 事件（`src/components/ScreenshotOverlay.tsx:544`），主窗口 `src/App.tsx:56` 监听后把图片追加到该笔记正文。
9. 关闭：`hide_overlay_window`（`src-tauri/src/screenshot.rs:149`）仅隐藏覆盖窗、复用不销毁。

## 五、核心能力详解

- **框选**：拖拽超过 5px 阈值判定为拖拽手势（`onPointerMove` 中判定，`src/components/ScreenshotOverlay.tsx:271`）；松手时选区宽度和高度均超过 12px 才保留为有效框选结果（`onPointerUp` 中判定，`:333`：拖动画出的框太小时视为误触丢弃）。两条阈值用途不同——5px 区分「点击」与「拖拽」，12px 保证框选结果有意义；Shift 锁定长宽比；选区尺寸实时以物理像素显示；方向键微移。
- **窗口识别**：悬停绿色高亮光标下「面积最小」的可见窗口（`hitWindow` `src/components/ScreenshotOverlay.tsx:106`）；单击整窗截取（`src/components/ScreenshotOverlay.tsx:319`）；窗口标题在悬停时按需懒加载（`get_window_title` `src-tauri/src/screenshot.rs:225`，单次 30ms 超时、绝不阻塞）。
- **长截图**：长截图入口点目标窗口 → `capture_window_full`（`src-tauri/src/screenshot.rs:672`）以 `PrintWindow + 程序化滚动 + 逐屏拼接`捕获完整内容，规避旧版 `PrintWindow(flag 2)` 的挂起问题；返回原始 PNG 字节由编辑态直接保存。
- **标注工具**：画笔 / 橡皮（`destination-out` 合成）/ 矩形 / 箭头 / 文字；`drawAnnotation` 实现矩形与箭头绘制（`src/components/ScreenshotOverlay.tsx:833`）；撤销用 `undoRef` 快照栈（`src/components/ScreenshotOverlay.tsx:462`）。
- **保存路径优化**：无标注且非长截图 → `crop_native_rgba`（`src-tauri/src/screenshot.rs:563`）直接拿原生 RGBA，不经过 PNG 编码；有标注 → `getImageData` 取像素级 RGBA（替代最慢的 `toBlob` 编码）。这是保存进入毫秒级的关键。

## 八、性能设计要点

- **秒开**：覆盖窗独立轻量入口，不加载主应用；预览图降采样为 JPEG，体积小、解码快。
- **无闪烁**：`start_screenshot` 改为「先同步捕获再显示覆盖窗（带冻结图）」，去掉透明→填充的二次刷新（`src-tauri/src/screenshot.rs:507`）。
- **毫秒级保存**：保存链路全程原生 RGBA 直通，跳过「前端 PNG 编码 → IPC base64 → Rust 再解码」最慢环节（见 `save_screenshot` `src-tauri/src/screenshot.rs:590` 注释）。
- **防卡顿**：`list_windows` 枚举时不读标题（`src-tauri/src/screenshot.rs:207`）；标题改悬停懒加载且 30ms 超时；`CAPTURING` 原子量防连按热键重复触发（`src-tauri/src/screenshot.rs:357`）。

### 11.1 坐标方案与 DPI（后端）

```rust
// src-tauri/src/screenshot.rs:1 实现功能：文件头注释，说明多显示器+DPI 一致性与坐标方案
// 关键：DPI 一致性。Tauri v2（TAO/WRY）在 Windows 上默认 Per-Monitor-DPI-Aware V2。
// 坐标方案（根治「边缘偏移 / 白边 / 截不到」）：
//   覆盖窗的尺寸/位置以「逻辑像素」设定（物理 = 逻辑 × scale_factor）；我们捕获的是
//   **覆盖窗自身的真实矩形**（outer_position/outer_size，物理像素），而非另算的虚拟桌面。
//   截图图在 WebView 内 object-fill 铺满覆盖窗，故：
//     图像像素(x,y) ≡ 窗口内 CSS 像素(x,y)   （预览图已按 scale_factor 降采样，1 CSS px = 1 图像 px）
//   映射退化为恒等，彻底消除偏移；且「捕获区域 == 覆盖窗显示区域」，边缘必然完整覆盖，无白边、无截不到。

// src-tauri/src/screenshot.rs:42 实现功能：系统 DPI 缩放比（如 150% 显示 → 1.5）
fn dpi_scale() -> f64 {
    unsafe {
        let dpi = winapi::um::winuser::GetDpiForSystem();
        if dpi == 0 { 1.0 } else { dpi as f64 / 96.0 }
    }
}

// src-tauri/src/screenshot.rs:57 实现功能：用 EnumDisplayMonitors 拿真实物理矩形并集（多屏覆盖）
fn virtual_desktop_rect() -> (i32, i32, i32, i32) {
    let mut data = (i32::MAX, i32::MAX, i32::MIN, i32::MIN);
    unsafe {
        winapi::um::winuser::EnumDisplayMonitors(
            std::ptr::null_mut(), std::ptr::null(),
            Some(monitor_enum_proc),
            &mut data as *mut _ as winapi::shared::minwindef::LPARAM,
        );
    }
    if data.0 == i32::MAX {
        let (x, y, w, h) = unsafe { ( /* 兜底 GetSystemMetrics SM_XVIRTUALSCREEN ... */ ) };
        return (x, y, w, h);
    }
    (data.0, data.1, data.2 - data.0, data.3 - data.1)
}
```

### 11.2 覆盖窗创建与复用

```rust
// src-tauri/src/screenshot.rs:100 实现功能：预创建/复用透明置顶覆盖窗（不重复创建 WebView2）
#[tauri::command]
pub fn create_overlay_window(app: tauri::AppHandle) -> Result<(), String> {
    use tauri::WebviewUrl;
    if app.get_webview_window("screenshot-overlay").is_some() {
        return Ok(()); // 已存在则复用，不重复创建
    }
    let scale = dpi_scale();
    let (rx, ry, rw, rh) = virtual_desktop_rect();
    let lw = rw as f64 / scale;
    let lh = rh as f64 / scale;
    let lx = rx as f64 / scale;
    let ly = ry as f64 / scale;
    let _win = tauri::WebviewWindowBuilder::new(
        &app, "screenshot-overlay", WebviewUrl::App("screenshot-overlay.html".into()),
    )
    .title("截图")
    .inner_size(lw, lh)
    .position(lx, ly)
    .decorations(false)
    .always_on_top(true)
    .skip_taskbar(true)
    .transparent(true)        // 透明：触发瞬间透出桌面，用户感知毫秒级响应，不闪白
    .visible(false)           // 预创建时隐藏；截图时由 start_screenshot 显示
    .resizable(false)
    .build()
    .map_err(|e| format!("创建截图窗口失败: {}", e))?;
    Ok(())
}

### 11.3 窗口枚举与标题懒加载

// src-tauri/src/screenshot.rs:157 实现功能：枚举其他进程可见窗口（物理像素矩形）
#[tauri::command]
pub fn list_windows() -> Result<Vec<WindowInfo>, String> {
    let mut windows: Vec<WindowInfo> = Vec::new();
    unsafe {
        winapi::um::winuser::EnumWindows(
            Some(enum_callback),
            &mut windows as *mut _ as winapi::shared::minwindef::LPARAM,
        );
    }
    windows.retain(|w| w.width > 8 && w.height > 8);
    Ok(windows)
}

// src-tauri/src/screenshot.rs:225 实现功能：悬停时按需懒加载单个窗口标题（30ms 超时、绝不阻塞截图路径）
#[tauri::command]
pub fn get_window_title(hwnd: u64) -> String {
    let hw: winapi::shared::windef::HWND = hwnd as winapi::shared::windef::HWND;
    let mut buf = [0u16; 512];
    let mut text_len: usize = 0;
    let sent = unsafe {
        winapi::um::winuser::SendMessageTimeoutW(
            hw, winapi::um::winuser::WM_GETTEXT, 511,
            buf.as_mut_ptr() as winapi::shared::minwindef::LPARAM,
            winapi::um::winuser::SMTO_ABORTIFHUNG, // 不带 NOTIMEOUTIFNOTHUNG，否则超时失效会挂起
            30, &mut text_len,
        )
    };
    if sent != 0 && text_len > 0 { String::from_utf16_lossy(&buf[..text_len]) } else { String::new() }
}
```

### 11.4 同步捕获整窗（BitBlt + 双份图）

```rust
// src-tauri/src/screenshot.rs:249 实现功能：将 (x,y,w,h) 区域从整屏 DC 拷出为 RgbaImage（物理像素，BGRA→RGBA）
fn capture_region(x: i32, y: i32, w: i32, h: i32) -> Result<RgbaImage, String> {
    if w <= 0 || h <= 0 { return Err("无效的截图区域".into()); }
    unsafe {
        let screen_dc = winapi::um::winuser::GetDC(std::ptr::null_mut());
        let mem_dc = winapi::um::wingdi::CreateCompatibleDC(screen_dc);
        let bitmap = winapi::um::wingdi::CreateCompatibleBitmap(screen_dc, w, h);
        let old = winapi::um::wingdi::SelectObject(mem_dc, bitmap as *mut winapi::ctypes::c_void);
        let _ok = winapi::um::wingdi::BitBlt(mem_dc, 0, 0, w, h, screen_dc, x, y, winapi::um::wingdi::SRCCOPY);
        // ... GetDIBits 取 BGRA → 转 RGBA ...
        let mut rgba = vec![0u8; buf.len()];
        for i in 0..((w as usize) * (h as usize)) {
            let j = i * 4;
            rgba[j] = buf[j + 2]; rgba[j + 1] = buf[j + 1]; rgba[j + 2] = buf[j]; rgba[j + 3] = 255;
        }
        match RgbaImage::from_raw(w as u32, h as u32, rgba) {
            Some(im) => Ok(im), None => Err("图像构造失败".into()),
        }
    }
}
```

#[tauri::command]
pub fn start_screenshot(app: tauri::AppHandle, _state: tauri::State<'_, std::sync::Mutex<ScreenshotData>>) -> Result<(), String> {
    create_overlay_window(app.clone())?;
    let overlay = app.get_webview_window("screenshot-overlay").ok_or_else(|| "覆盖窗缺失".to_string())?;
    // 防重入：正在捕获或已可见则忽略

        return Ok(());
    }

    let _ = overlay.hide();
    let (rx, ry, rw, rh) = virtual_desktop_rect();

    let full = match capture_region(ox - rx, oy - ry, ow, oh) { Ok(f) => f, Err(e) => { /* ... */ return Err(e); } };

    let full_dyn = DynamicImage::ImageRgba8(full);
    // 预览：降采样到逻辑分辨率 + JPEG（编码快、体积小，是「截图秒开」的关键）
    let nw = ((ow as f64) / scale).round().max(1.0) as u32;
    let nh = ((oh as f64) / scale).round().max(1.0) as u32;
    let preview_dyn = if nw < (ow as u32) || nh < (oh as u32) {
        full_dyn.resize(nw, nh, FilterType::CatmullRom)
    } else { full_dyn };
    let preview_rgb = preview_dyn.to_rgb8();
    let mut pv_buf = Cursor::new(Vec::new());
    DynamicImage::ImageRgb8(preview_rgb).write_to(&mut pv_buf, ImageFormat::Jpeg).map_err(|e| format!("预览编码失败: {}", e))?;
    { let mut slot = SHOT.lock().expect("截图状态锁失败");
        *slot = Some(Shot { preview: pv_buf.into_inner(), raw, native_w: ow as u32, native_h: oh as u32, native_ox: ox, native_oy: oy }); }
    let windows = list_windows().unwrap_or_default();
    let note_id = app.state::<std::sync::Mutex<ScreenshotData>>().lock().map(|s| s.note_id.clone()).unwrap_or_default();
    let payload = json!({ "ox": ox, "oy": oy, "scale": scale, "windows": windows, "noteId": note_id });
    let _ = overlay.emit("screenshot-start", payload);  // 捕获完成后再显示覆盖窗（已是冻结图，无二次刷新）

    let _ = overlay.set_focus();

// src-tauri/src/screenshot.rs:979 实现功能：解析快捷键字符串（如 "Ctrl+Shift+S"）为 tauri global_shortcut::Shortcut
fn parse_shortcut(s: &str) -> Result<Shortcut, String> {
    let s = s.replace(' ', "");
    if s.is_empty() { return Err("空快捷键".to_string()); }
    let parts: Vec<&str> = s.split('+').collect();
    let key = *parts.last().unwrap();
    let mut modifiers = Modifiers::empty();
    for p in &parts[..parts.len() - 1] {
        match p.to_uppercase().as_str() {
            "CTRL" | "CONTROL" => modifiers |= Modifiers::CONTROL,
            "SHIFT" => modifiers |= Modifiers::SHIFT,
            "ALT" => modifiers |= Modifiers::ALT,
            "META" | "WIN" | "SUPER" | "CMD" | "WINDOWS" => modifiers |= Modifiers::SUPER,
            _ => return Err(format!("不支持的修饰键: {}", p)),
        }
    }
    let code = parse_code(key)?;
    Ok(Shortcut::new(Some(modifiers), code))
}

// src-tauri/src/screenshot.rs:1051 实现功能：设置面板改键（先持久化文件，再注销旧键、注册新键，失败回退旧键）
#[tauri::command]
pub fn set_screenshot_shortcut(app: tauri::AppHandle, shortcut: String) -> Result<(), String> {
    let new = parse_shortcut(&shortcut)?;
    let old = app.state::<std::sync::Mutex<ScreenshotData>>().lock().ok()
        .map(|s| s.shortcut.clone()).unwrap_or_else(|| "Ctrl+Shift+S".to_string());
    write_screenshot_shortcut(&app, &shortcut)?;  // 先持久化文件，再注册（保证下次启动读新值）
    if let Ok(mut s) = app.state::<std::sync::Mutex<ScreenshotData>>().lock() { s.shortcut = shortcut.clone(); }
    if let Ok(old_sc) = parse_shortcut(&old) { let _ = app.global_shortcut().unregister(old_sc); }
    app.global_shortcut().register(new).map_err(|e| {
        if let Ok(old_sc) = parse_shortcut(&old) { let _ = app.global_shortcut().register(old_sc); }
        format!("注册失败（已回退原热键）: {}", e)
    })?;
    Ok(())
}
